\hypertarget{md_external_2libs_2pgbar_2README_autotoc_md297}{}\doxysubsubsection{\texorpdfstring{Does updating the progress bar slow down the program?}{Does updating the progress bar slow down the program?}}\label{md_external_2libs_2pgbar_2README_autotoc_md297}
No, as mentioned in the Features section, updating the progress bar has {\itshape essentially} zero overhead.

With {\ttfamily O2/\+O3} optimizations enabled, the performance overhead of the second iteration in the code below will {\itshape approach} that of the first iteration\mbox{[}\texorpdfstring{$^\wedge$}{\string^}1\mbox{]}.

\mbox{[}\texorpdfstring{$^\wedge$}{\string^}1\mbox{]}\+: In practice, the overhead of updating the progress bar only adds a single function call and some branching cost.
